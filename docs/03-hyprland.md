# 03 ¬∑ Hyprland

Hyprland is the Wayland compositor and tiling window manager ‚Äî it draws every
window, handles blur/animations, and owns all global keyboard shortcuts. HyDE
configures most of it; you customize through a few designated files.

## File layout

```
.config/hypr/
‚îú‚îÄ‚îÄ hyprland.conf       # entrypoint ‚Äî sources everything else (HyDE-owned top marker)
‚îú‚îÄ‚îÄ userprefs.conf      # ‚úÖ YOUR personal tweaks (input, gestures, misc)
‚îú‚îÄ‚îÄ keybindings.conf    # ‚úÖ keyboard shortcuts
‚îú‚îÄ‚îÄ windowrules.conf    # ‚úÖ per-app rules (opacity, float, PiP)
‚îú‚îÄ‚îÄ monitors.conf       # ‚úÖ display layout (generated by nwg-displays)
‚îú‚îÄ‚îÄ workspaces.conf     # workspace rules
‚îú‚îÄ‚îÄ animations.conf     # picks an animation preset from animations/
‚îú‚îÄ‚îÄ animations/         # animation presets (classic, fast, minimal, diablo, ...)
‚îú‚îÄ‚îÄ workflows.conf      # picks a workflow preset
‚îú‚îÄ‚îÄ workflows/          # performance/behavior presets (gaming, editing, powersaver...)
‚îú‚îÄ‚îÄ shaders.conf        # screen shaders (blue-light filter, grayscale, vibrance...)
‚îú‚îÄ‚îÄ shaders/            # the .frag shader sources
‚îú‚îÄ‚îÄ hyprlock.conf       # lockscreen, picks a layout from hyprlock/
‚îú‚îÄ‚îÄ hyprlock/           # lockscreen layouts (HyDE, SF Pro, IBM Plex, ...)
‚îú‚îÄ‚îÄ hypridle.conf       # idle behavior (dim, lock, suspend)
‚îú‚îÄ‚îÄ hyprsunset.conf     # night-light / color temperature
‚îî‚îÄ‚îÄ pyprland.toml       # pyprland plugins (e.g. the dropdown terminal "console")
```

`hyprland.conf` starts with a HyDE marker line you must not remove:

```
$HYDE_HYPRLAND=set #! Do not remove - HyDE marker to prevent file overwrite
```

It then `source`s `keybindings.conf`, `windowrules.conf`, `monitors.conf`, and
`userprefs.conf`. **`userprefs.conf` is where your personal Hyprland settings go** ‚Äî
it's sourced last, so it wins.

## userprefs.conf ‚Äî your personal layer

Currently mostly commented scaffolding. Useful things to uncomment here:

```ini
input {
    kb_layout = us
    numlock_by_default = true
    touchpad { natural_scroll = no }
}
gestures {
    workspace_swipe = true          # 3-finger swipe between workspaces
    workspace_swipe_fingers = 3
}
misc {
    enable_swallow = true           # terminal "swallows" GUI apps it launches
    swallow_regex = (kitty|Alacritty|ghostty)
}
```

## Keybindings

The keybinding file uses HyDE's `bindd` ("bind with description") syntax. The `$d`
variable groups binds for the on-screen cheat-sheet (`Super+/`). `$mainMod` is the
**Super** key (Windows key).

## Window rules

`windowrules.conf` defines per-application behavior. The two big categories:

**Opacity** ‚Äî makes specific apps translucent (works with blur for the frosted
look). Format: `windowrule = opacity <active> $& <inactive> $& <fullscreen>,match:class ^(app)$`.

```ini
windowrule = opacity 0.80 $& 0.80 $& 1, match:class ^(kitty)$      # terminal
windowrule = opacity 0.90 $& 0.90 $& 1, match:class ^(firefox)$
windowrule = opacity 0.80 $& 0.80 $& 1, match:class ^([Cc]ode)$
windowrule = opacity 0.70 $& 0.70 $& 1, match:class ^([Ss]potify)$
```

‚öÝÔ∏è **This is where kitty's transparency actually comes from** ‚Äî not from kitty's own
config. The `1` at the end keeps fullscreen windows opaque (so video/games aren't
washed out). To make the terminal more/less see-through, change the `0.80` here.

**Float / PiP** ‚Äî forces certain apps (Signal, image viewers, dialogs) to float
instead of tile. There's also a tagged Picture-in-Picture rule that pins a small
floating video to the bottom-right.

**Layer rules** (bottom of the file) ‚Äî enable blur on overlays (rofi,
notifications, logout dialog) so they get the frosted-glass effect.

## Monitors

`monitors.conf` is generated by `nwg-displays` (a GUI). Current config:

```ini
monitor=HDMI-A-2,1920x1080@165.0,0x0,1.0
#       output    resolution@refresh  position scale
```

To change resolution/refresh/position, run `nwg-displays` rather than editing by
hand ‚Äî it regenerates this file (and a matching workspace map).

## Animations, workflows, shaders

These are *preset systems*: a top-level `.conf` selects one file from a sibling
directory.

- **Animations** (`Super+Shift+Y`) ‚Äî `animations.conf` sources one of
  `animations/*.conf` (classic, fast, minimal-1/2, diablo-1/2, vertical, disable‚Ä¶).
  Disable for max performance.
- **Workflows** ‚Äî `workflows.conf` selects a behavior preset from `workflows/`
  (default, gaming, editing, powersaver, snappy). E.g. `gaming.conf` disables blur
  and animations for FPS; `powersaver.conf` reduces blur passes.
- **Shaders** (`hyde-shell` / config) ‚Äî `shaders/` holds GLSL fragment shaders:
  blue-light filter, grayscale, color-vision (colorblind aid), vibrance, OLED
  saver, invert. `shaders.conf` picks the active one.

## Idle, lock, night light

- `hypridle.conf` ‚Äî what happens when idle (dim ‚Üí lock ‚Üí suspend timers).
- `hyprlock.conf` + `hyprlock/` ‚Äî the lockscreen; `Super+Shift+U` swaps layouts
  (HyDE, SF Pro, IBM Plex, Anurati, Arfan on Clouds‚Ä¶).
- `hyprsunset.conf` ‚Äî automatic warm color temperature at night.
