# 09 · Waybar & Dunst

The desktop's status bar and its notifications. Both are heavily HyDE/wallbash
managed, so this repo tracks only the *override* layer for each.

## Waybar (status bar)

Waybar is the bar across the top of the screen (clock, workspaces, tray, system
stats). HyDE generates Waybar's main config and style from layout presets via
`wbarconfgen` — so the bulk of Waybar is **generated, not tracked**.

### What's tracked vs generated

| File | Status |
|---|---|
| `.config/waybar/user-style.css` | ✅ yours — the only tracked Waybar file |
| `.config/waybar/config.jsonc` | ⛔ generated by `wbarconfgen` |
| `.config/waybar/style.css` | ⛔ generated (imports your `user-style.css`) |
| `.config/waybar/colors.css` | ⛔ wallbash output |

### user-style.css — your override layer

The generated `style.css` `@import`s `user-style.css` *last*, so any CSS you put
there wins. This is where you tweak module spacing, fonts, corner radius, custom
module colors — without touching generated files.

```css
/* example overrides you might add to user-style.css */
#clock { font-weight: bold; padding: 0 12px; }
#workspaces button.active { border-radius: 8px; }
```

### Switching Waybar layouts

HyDE ships multiple Waybar layouts. Cycle them live:

| Keys | Action |
|---|---|
| `Super+Alt+↑` | Next Waybar layout |
| `Super+Alt+↓` | Previous Waybar layout |

Each layout regenerates `config.jsonc`/`style.css` — your `user-style.css`
overrides ride along on top of whichever layout is active.

🔗 Waybar wiki: https://github.com/Alexays/Waybar/wiki

## Dunst (notifications)

Dunst draws popup notifications (volume changes, app alerts, screenshots). Its
colors are wallbash-driven so notifications match the active theme.

### Files

```
.config/dunst/
├── dunst.conf    # the active config dunst reads
└── dunstrc       # config source (HyDE/wallbash conventions)
```

Colors (background/foreground/frame) are filled by wallbash from the current
wallpaper, so a notification's accent matches your desktop. The **layout** —
geometry, font, timeouts, urgency behavior, action buttons — is what you'd
hand-edit.

### Things you can safely tune

```ini
[global]
    width = 350
    origin = top-right
    offset = 10x50
    font = CaskaydiaCove Nerd Font 10
    frame_width = 2
    corner_radius = 12          # rounded corners to match the rice

[urgency_low]    { timeout = 4 }
[urgency_normal] { timeout = 8 }
[urgency_critical] { timeout = 0 }   # never auto-dismiss
```

⚠️ If colors are wallbash-templated in these files, leave the color lines alone and
only change geometry/font/timeout. Changing the wallpaper recolors notifications;
editing color hex here would be overwritten.

### Blur on notifications

Notifications get the frosted-glass look from a Hyprland **layer rule** (in
[`windowrules.conf`](03-hyprland.md#window-rules)):

```ini
layerrule = blur true, match:namespace notifications
```

🔗 Dunst docs: https://dunst-project.org/documentation/
